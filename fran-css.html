Text Headers

Note: Do not use a specific header tag just because you like the color, font size, etc. Headers should organize content on the page in number order and should contain content below the header that is relevant to the header. Think of headers how you would organize notes in school. Maybe you'd organize them first by the name of the chapter (h1), then by chapter sections (h2), then by key points for each chapter section (h3), and so on.

H1

H1 tags are typically only used within hero headers. Jump to heros.

H2

Franciscan H2 tags are tyically styled by using the class root-h2.

H2 tags should be used for headers after use of an H1 tag, for top level organization of content on a page.

.root-h2 { text-align: center; color: var(--fran-red); font-size: 1.7em; line-height: 1.2em; font-family: 'bennet-banner'; font-weight: 'bold'; margin-bottom: 1em; }

Other classes and inline styles can be used to alter the h2 text's styling.

H3

Franciscan H3 tags are tyically styled by using the class root-h3.

H3 tags should be used for organization of content under h2 tags.

.root-h2 { color: var(--fran-green); font-size: 1.5em; line-height: 1.2em; font-family: 'objektiv-mk3'; font-weight: 'bold'; margin-bottom: 1em; }

Other classes and inline styles can be used to alter the h3 text's styling.

Other Headers

Other headers (as of the date this page has been updated) typically rely on the Wordpress theme's settings applied to those headers.

See the Wordpress theme backend for styles.

Buttons

.root-lowercase-btn { background-color: var(--root-lowercase-btn-background-color); --root-lowercase-btn-background-color: #fbb555; color: var(--root-lowercase-btn-color); --root-lowercase-btn-color: #2b2b2b; border: 1px solid transparent; border-color: var(--root-lowercase-btn-border-color); --root-lowercase-btn-border-color: transparent; border-radius: 25px; font-size: 1em; font-family: 'Montserrat'; padding: .75em 1em; font-weight: 600; text-decoration: none; cursor: pointer; transition: 0.2s; display: block; width: fit-content; margin: 10px auto; } .root-lowercase-btn:hover { background-color: var(--root-lowercase-btn-background-color); --root-lowercase-btn-background-color: #fbb555; color: var(--root-lowercase-btn-color); --root-lowercase-btn-color: #2b2b2b; box-shadow: 0 0 5px #fbb5555c; }

There's really only one button class used in regular page development: root-lowercase-btn.

Button

The 'root lowercase button' is named such since some original designs had multiple types of buttons, warranting the use of 'lowercase-root-btn' as opposed to some obsolete classes, i.e. 'site-lrg-btn' (within the Franciscan Giving subdomain).

At times, this class might be used in an environment where only inline editing of styles is available (as opposed to custom classes). This is why the color variables exist:

  • --root-lowercase-btn-background-color

  • --root-lowercase-btn-color

  • --root-lowercase-btn-background-color-hover

  • --root-lowercase-btn-color-hover

Using these styles inline allows the hover state to be changed regardless of the amount of CSS freedom you have. For example:

<a href="#" class="root-lowercase-btn" style="--root-lowercase-btn-background-color: #21412a; --root-lowercase-btn-color: white; --root-lowercase-btn-background-color-hover: purple; --root-lowercase-btn-color-hover: rgb(239, 221, 150);">Test Button</a>
